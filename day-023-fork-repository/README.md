# Fork a Git Repository Using Gitea

## 1. What is the Challenge?

The objective is to fork an existing Git repository using the **Gitea** web interface. A new developer named **jon** needs a personal copy of the repository **sarah/story-blog** so development can begin without affecting the original repository.

# 2. Required Technology to Solve It

- **Gitea**
- **Git**
- **Web Browser**

### Useful Information

Login Credentials:

```text
Username: jon
Password: Jon_pass123
```

Repository to Fork:

```text
sarah/story-blog
```

Expected Result:

```text
jon/story-blog
```

# 3. How to Solve It

### Step 1

Open the **Gitea UI** from the KodeKloud lab.

### Step 2

Sign in using the provided credentials.

### Step 3

Search for the repository:

```text
sarah/story-blog
```

### Step 4

Open the repository and click the **Fork** button.

### Step 5

Verify that the repository has been forked successfully under the **jon** account.

# 4. Main Takeaways

- Learned how to fork an existing Git repository
- Understood the purpose of repository forking for collaborative development
- Created a personal copy of an existing repository without modifying the original
- Verified the fork through the Gitea web interface

# 5. Conclusion

This task demonstrates how developers can create their own working copy of an existing repository using Gitea's **Fork** feature. Forking enables independent development while preserving the original repository, making it a common workflow in collaborative software development.
